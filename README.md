# Система контроля кормления рыб

Система автоматического мониторинга кормления рыб на основе компьютерного зрения. Анализирует видеопоток с камеры, считает гранулы корма в реальном времени и уведомляет оператора о нештатных ситуациях.

## Архитектура

```
server/   — ML-сервер (Python, FastAPI, YOLOv8)
client/   — Десктопное приложение (C#, Avalonia UI)  [в разработке]
```

ML-сервер и клиент взаимодействуют исключительно через REST API.

## Серверная часть

### Стек

- Python 3.10+
- FastAPI — REST API
- YOLOv8 (ultralytics) — детекция гранул
- SQLAlchemy + MySQL — хранение событий и настроек
- python-jose — JWT-аутентификация

### Установка

```bash
cd server
pip install -r requirements.txt
```

Создать файл `.env`:

```env
DB_URL=mysql+pymysql://user:password@localhost:3306/dronevision
SECRET_KEY=your-secret-key
```

Положить обученные веса модели в `server/weights/best.pt`.

### Запуск

```bash
cd server
uvicorn main:app --host 0.0.0.0 --port 8000
```

После запуска автоматически создаётся администратор по умолчанию: `admin / admin`.

Документация API доступна по адресу: `http://localhost:8000/docs`

### Структура

```
server/
├── main.py               — точка входа, инициализация приложения
├── config.py             — настройки через переменные окружения
├── database.py           — модели БД и подключение
├── models/
│   ├── types.py          — базовые типы (Detection)
│   ├── detector.py       — обёртка над YOLOv8
│   └── counter.py        — алгоритм подсчёта гранул и расчёта интенсивности
├── api/
│   ├── auth.py           — авторизация, управление пользователями
│   ├── stream.py         — приём видео, запуск/остановка анализа
│   ├── events.py         — журнал событий кормления
│   ├── stats.py          — статистика за период
│   ├── logs.py           — журнал ошибок
│   └── settings_api.py   — конфигурация системы
├── schemas/
│   └── schemas.py        — Pydantic-схемы запросов и ответов
└── tests/
    └── test_counter.py   — тесты алгоритма подсчёта
```

### API

| Метод | Endpoint | Описание |
|-------|----------|----------|
| POST | `/auth/login` | Получить JWT-токен |
| GET | `/auth/me` | Текущий пользователь |
| GET | `/auth/users` | Список пользователей (admin) |
| POST | `/auth/users` | Создать пользователя (admin) |
| DELETE | `/auth/users/{id}` | Удалить пользователя (admin) |
| POST | `/stream/upload` | Загрузить MP4/AVI и запустить анализ |
| POST | `/stream/stop` | Остановить анализ |
| GET | `/stream/status` | Статус обработки |
| GET | `/events` | Журнал событий кормления |
| PATCH | `/events/{id}/comment` | Добавить комментарий к событию |
| GET | `/stats` | Статистика за период |
| GET | `/logs` | Журнал ошибок |
| GET | `/settings` | Текущие настройки |
| PATCH | `/settings` | Обновить настройки |

### Алгоритм подсчёта интенсивности

Интенсивность рассчитывается по скользящему окну (по умолчанию 5 секунд): суммируются все гранулы, обнаруженные в кадрах внутри окна, и делятся на фактическую длительность окна. Это даёт сглаженный показатель потока корма без шумовых пиков.
